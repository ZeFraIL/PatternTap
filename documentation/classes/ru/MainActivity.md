# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** `Activity`
*   **Назначение класса в приложении:** Это сердце приложения "PatternTap". Оно выступает в роли «Режиссера» или «Контроллера» игры. Класс отвечает за всё: от настройки визуальной сетки и кнопок до управления игровой логикой (генерация последовательностей, проверка пользовательского ввода и отслеживание счета).
*   **Как он взаимодействует с другими компонентами:** Являясь единственным Activity в проекте, он взаимодействует с системой Android для управления своим жизненным циклом и использует внутренние компоненты, такие как `Handler` для работы с задержками и `Random` для создания случайных игровых ситуаций.

## 3. Методы класса

### Имя метода: `onCreate(Bundle savedInstanceState)`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `savedInstanceState` (`Bundle`) - Сохраненное состояние для восстановления.
*   **Что делает:** 
    1.  Настраивает визуальный макет.
    2.  Инициализирует элементы интерфейса (кнопки, ползунок, текстовые поля).
    3.  Устанавливает слушателей (обработчиков) для кнопок и ползунка скорости.
    4.  Вызывает `setupGrid()` для подготовки игрового поля.
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Важно понимать:** Это точка входа. Если компонент не инициализирован здесь, приложение упадет с ошибкой `NullPointerException`.

### Имя метода: `setupGrid()`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает:** 
    1.  Удаляет существующие кнопки из `gridLayout`.
    2.  Устанавливает количество строк и столбцов на основе `gridSize`.
    3.  В цикле создает новые объекты `Button`, задает их размер, отступы и теги (от 0 до N).
    4.  Добавляет каждую кнопку в список `buttons` и в макет.
*   **Когда вызывается:** При старте приложения, при перезапуске игры или при повышении уровня.
*   **Важно понимать:** Метод динамически строит интерфейс. Использование тегов (tags) критически важно, так как это помогает позже определить, какая именно кнопка была нажата.

### Имя метода: `startNewRound()`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает:** 
    1.  Генерирует новый случайный паттерн.
    2.  Сбрасывает `waveIndex`.
    3.  Запускает первый шаг анимации волны через `handler`.
*   **Когда вызывается:** При нажатии кнопки "Restart" или при завершении предыдущей последовательности.

### Имя метода: `waveRunnable` (Логика в `run()`)
*   **Тип:** `private final Runnable`
*   **Возвращаемое значение:** `void`
*   **Что делает:** 
    1.  Проверяет, не пропустил ли игрок предыдущий шаг ("Too slow!").
    2.  Сбрасывает цвет всех кнопок на светло-серый.
    3.  Проверяет условия повышения уровня (каждые 5 очков) или победы (25 очков).
    4.  Подсвечивает следующую кнопку в `pattern` желтым цветом.
    5.  Планирует свой следующий запуск через `waveSpeed` миллисекунд.
*   **Когда вызывается:** Повторяющееся действие через `handler.postDelayed`, пока игра активна.
*   **Важно понимать:** Это "сердцебиение" игры. Метод контролирует темп происходящего.

### Имя метода: `onClick(View v)`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `v` (`View`) - конкретная кнопка, на которую нажали.
*   **Что делает:** 
    1.  Проверяет, разрешено ли нажатие (игра должна быть активна, а "волна" в процессе).
    2.  Сравнивает тег нажатой кнопки с ожидаемым индексом из `pattern`.
    3.  Если верно: Увеличивает счет, обновляет UI и отмечает шаг как угаданный (кнопка становится зеленой).
    4.  Если неверно: Кнопка становится красной и выводится сообщение "Wrong!".
*   **Когда вызывается:** Каждый раз, когда пользователь нажимает на кнопку в сетке.

## 4. Жизненный цикл (Activity)
*   **`onCreate()`**: Вызывается при первом создании Activity. Инициализирует всё необходимое.
*   **Примечание**: Другие методы, такие как `onPause()` или `onDestroy()`, явно не переопределены, что означает, что игра не останавливается автоматически при сворачивании приложения (это можно улучшить!).

---

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `GridLayout` (поле), `TextView` (счет/метки), `Button` (управление), `SeekBar` (скорость).
*   **Связь с кодом:** Инициализация через `findViewById(R.id.имя_id)`.
*   **События:** 
    *   `setOnClickListener`: Используется для кнопок управления и кнопок в сетке.
    *   `OnSeekBarChangeListener`: Используется для обновления `waveSpeed` в реальном времени при движении ползунка.

---

## 6. Взаимодействие с другими компонентами
*   **Intent:** Не используются (приложение состоит из одного экрана).
*   **Передача данных:** Все данные остаются внутри экземпляра `MainActivity`.

---

## 7. Общая логика класса
Класс следует циклу: **Генерация -> Показ -> Проверка -> Повтор**.
1.  **Старт**: Игрок нажимает Restart. `setupGrid()` строит UI.
2.  **Волна**: `Runnable` работает по таймеру. Он берет кнопку из случайного списка и подсвечивает её желтым.
3.  **Взаимодействие**: Игрок должен нажать на эту кнопку до того, как `Runnable` перейдет к следующему шагу.
4.  **Обратная связь**: Приложение дает визуальную (цвета) и текстовую (Toast) обратную связь.
5.  **Прогресс**: Каждые 5 правильных ответов сетка увеличивается, усложняя задачу.

## 8. Упрощенное объяснение
**Представьте себе маяк.** 
Смотритель маяка (`Runnable`) по очереди светит фонарем на разные камни в море в определенном порядке. Вы — моряк, который должен указать на тот же камень, который только что осветил смотритель. Если вы медлите, смотритель переходит к следующему камню, и вы упускаете свой шанс. По мере того как вы справляетесь, смотритель светит быстрее, а со временем в море появляется всё больше камней!

**Аналогия из жизни:** 
Это похоже на классическую игру «Саймон говорит», но вместо того, чтобы запоминать длинную последовательность и повторять её в конце, вы должны «ловить» сигнал прямо в момент его появления.
