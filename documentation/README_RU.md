## 🎯 Цель проекта
• **Какую задачу решает приложение:** Развитие зрительной памяти и скорости реакции через интерактивную игру.  
• **Почему эта задача важна:** Геймификация когнитивных тренировок помогает поддерживать активность мозга.  
• **Целевая аудитория:** Пользователи всех возрастов, желающие улучшить концентрацию и память.  

---

## 📌 Требования к приложению
### Функциональные требования
• Генерация случайных последовательностей (паттернов) на сетке.  
• Динамическое изменение размера сетки (от 3x3 до 5x5) при наборе очков.  
• Возможность настройки скорости игры через SeekBar.  
• Система подсказок и отслеживание текущего счета.  

### Нефункциональные требования
• **Производительность:** Плавная анимация подсветки кнопок без задержек.  
• **Удобство использования:** Простой интерфейс, интуитивно понятное управление.  
• **Надёжность:** Стабильная работа игрового цикла, предотвращение ложных нажатий.  

---

## 🧠 Общая архитектура
• **Выбранный подход:** MVC (Model-View-Controller).  
• **Почему выбран именно он:** Для небольшого игрового проекта этот подход позволяет быстро реализовать логику, где MainActivity управляет состоянием (Model) и обновляет интерфейс (View).  
• **Основные компоненты системы:**  
  – `MainActivity`: управление жизненным циклом и логикой.  
  – `GridLayout`: динамическое отображение игрового поля.  
  – `Handler`: обеспечение временных интервалов игры.  

---

## 🧩 UML-диаграмма
`[MainActivity] –> [Handler] (управление временем)`  
`[MainActivity] –> [GridLayout] (отображение кнопок)`  
`[MainActivity] –> [Random] (генерация паттерна)`  

**Описание пакетов:**  
Проект использует плоскую структуру пакетов (`zeev.fraiman.patterntap`), что упрощает навигацию в рамках небольшого приложения. При расширении логично выделить пакеты `.ui`, `.logic` и `.data`.

---

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:** Основной контроллер игры.  
**Ответственность:** Инициализация UI, обработка нажатий, управление игровым циклом, подсчет очков.  
**Основные методы:**  
- `onCreate()` — настройка интерфейса и слушателей.  
- `setupGrid()` — динамическое создание кнопок в сетке.  
- `startNewRound()` — запуск новой волны паттерна.  
- `waveRunnable` — основной цикл анимации и проверки времени.  
- `onClick()` — обработка пользовательского ввода и сравнение с эталоном.  
**Взаимодействие с другими классами:** Использует `Handler` для планирования задач и `Random` для логики.

## 🔄 Диаграмма работы приложения (Сценарий)
1. Пользователь нажимает "Restart".  
2. Система генерирует случайную последовательность кнопок.  
3. `Handler` поочередно подсвечивает кнопки желтым цветом.  
4. Пользователь должен успеть нажать на подсвеченную кнопку до следующего шага.  
5. При правильном нажатии начисляются очки, при ошибке или пропуске — уведомление.  
6. При достижении порога очков уровень сложности (сетка) увеличивается.

• **Принципы:**

  – **Логичность:** Цветовая индикация (зеленый — успех, красный — ошибка).  
  – **Доступность:** Крупные кнопки для удобства нажатия.

## ⚙️ Работа с потоками
• **Используются:** `Handler` (с `Looper.getMainLooper()`).  
• **Почему выбран этот способ:** Это стандартный механизм Android для работы с задержками в UI-потоке без блокировки интерфейса.  
• **Предотвращение проблем:**  
  – **ANR:** Все длительные операции (генерация) минимальны по времени.  
  – **Утечки памяти:** Использование `handler.removeCallbacks()` при остановке игры.

---

## 💾 Работа с данными
• **Где хранятся:** В оперативной памяти (переменные класса).  
• **Почему выбран этот способ:** Данные сессионные и не требуют сохранения между перезапусками приложения на данном этапе.  
• **Обеспечение корректности:** Строгая типизация и сброс состояния при перезапуске.
